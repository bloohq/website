title: Disparadores
description: 
---

Um disparador é um evento que aciona uma automação. Por exemplo, um disparador pode ser a conclusão de um registro, uma mudança no responsável por um registro, ou uma alteração em um campo personalizado. Quando essa condição predefinida é atendida, a automação é ativada.

Estes são todos os possíveis disparadores de automação no Blue:

- **Um Novo Registro é Criado**: Dispara quando um novo registro é adicionado ao projeto. Útil para iniciar ações de acompanhamento ou notificações relacionadas a novas tarefas ou entradas. Isso também dispara quando há um [novo envio de formulário](/pt/docs/projects/forms).
- **Um Registro é Movido para Outra Lista**: Ativa quando um registro é transferido de uma lista para outra dentro do projeto, indicando uma mudança em seu estágio ou status.
- **Um Registro é Marcado como Concluído**: Dispara quando o status de um registro é atualizado para 'concluído', sinalizando o fim de uma tarefa ou processo.
- **Um Registro é Marcado como Incompleto**: Ativa quando um registro anteriormente marcado como concluído é revertido para um status incompleto.
- **Alguém é Designado para um Registro**: Dispara quando um novo responsável é adicionado a um registro, indicando sua responsabilidade por essa tarefa ou item.
- **Alguém é Removido da Designação de um Registro**: Ativa quando um responsável é removido de um registro, significando uma mudança na responsabilidade da tarefa.
- **Data de Vencimento Muda**: Dispara quando a data de vencimento de um registro é alterada, o que pode ser crítico para tarefas sensíveis ao tempo.
- **Data de Vencimento é Removida**: Ativa quando a data de vencimento de um registro é apagada, possivelmente indicando uma mudança nas prioridades ou cronogramas.
- **Data de Vencimento Expirou**: Dispara quando a data de vencimento de um registro passou sem conclusão, destacando possíveis atrasos ou negligências. Esta é uma excelente automação para gestores, pois eles podem sinalizar registros/tarefas para si mesmos se elas ficarem atrasadas.
- **Tag é Adicionada ao Registro**: Ativa quando uma nova tag é adicionada a um registro, útil para categorização e acompanhamento de mudanças nos atributos da tarefa.
- **Tag é Removida do Registro**: Dispara quando uma tag é removida de um registro, indicando uma mudança em sua categorização ou propriedades.
- **Todas as Listas de Verificação são Marcadas como Concluídas**: Ativa quando cada item na lista de verificação de um registro é marcado, indicando a conclusão de todas as subtarefas.
- **Qualquer Lista de Verificação é Marcada como Incompleta**: Dispara quando qualquer item na lista de verificação de um registro é marcado como incompleto, sinalizando subtarefas inacabadas.
- **Um Registro é Movido/Copiado de Outro Projeto**: Ativa quando um registro é trazido para o projeto atual de um projeto diferente, útil para rastrear fluxos de trabalho entre projetos. O disparador se aplica tanto quando o registro é movido ou copiado manualmente por um usuário, quanto quando é transferido automaticamente através de outra automação, fornecendo uma maneira abrangente de rastrear atividades entre projetos.
- **Um Campo Personalizado é Adicionado a um Registro**: Dispara quando dados são escritos em um campo personalizado específico em um registro, indicando uma mudança ou adição de informação específica. Isso funciona apenas com caixas de seleção, campos personalizados de seleção única e múltipla.
- **Um Campo Personalizado é Removido de um Registro**: Ativa quando dados são removidos de um campo personalizado específico em um registro, significando uma modificação na estrutura de informações do registro.
- **Uma Cor é Adicionada a um Registro**: Dispara quando uma cor predefinida é atribuída a um registro, indicando prioridade, progresso ou categorização. Por exemplo, adicionar uma cor vermelha para tarefas de alta prioridade ou uma cor verde para itens em progresso.
- **Uma Cor é Removida de um Registro**: Dispara quando uma cor predefinida é removida de um registro, sinalizando uma mudança no status ou prioridade. Por exemplo, remover uma cor vermelha de uma tarefa quando ela não é mais de alta prioridade ou remover uma cor verde quando uma tarefa é concluída.
- **Botão de Campo Personalizado Clicado**: Dispara quando um usuário clica em um botão de campo personalizado em um registro. Isso permite fluxos de trabalho interativos onde os usuários podem acionar manualmente automações clicando em botões designados dentro dos registros.
