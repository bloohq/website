O campo personalizado de Arquivo permite carregar e anexar arquivos diretamente aos registros no Blue. Praticamente qualquer tipo de arquivo é suportado, incluindo documentos, planilhas, imagens, PDFs e muito mais.

![Arquivo 1](/docs/File_1.png "Arquivo 1")

Uploads de arquivos estruturados garantem que todos tenham visibilidade sobre os arquivos relacionados

Cada campo de arquivo permite carregar múltiplos arquivos, com cada arquivo de até 5GB de tamanho. Arquivos carregados, como imagens, PDFs e documentos do Word, podem ser visualizados se o tipo de arquivo for suportado. Todos os usuários do projeto podem baixar os arquivos anexados para acessar informações e conteúdo relevantes do registro.

![Arquivo 2](/docs/File_2.png "Arquivo 2")

Usuários podem visualizar e baixar arquivos no campo personalizado de Arquivo

Arquivos carregados são compartilhados em todo o projeto na [aba 'Arquivos'](/pt/docs/projects/file-management). No entanto, funções de usuário personalizadas com acesso restrito a registros não verão esses arquivos. A aba Arquivos do projeto suporta criação de pastas para organizar arquivos, mas os arquivos não podem ser organizados nessas pastas de dentro do próprio campo personalizado.

::callout

O histórico de versões de arquivos não é mantido — novos uploads não substituem arquivos existentes. Arquivos também não são incluídos em relatórios de exportação CSV.

## Integração com Formulários

O campo personalizado de Arquivo [integra-se perfeitamente com Formulários](/pt/docs/projects/forms), permitindo que usuários externos carreguem arquivos ao enviar informações. Por exemplo, candidatos podem anexar currículos e cartas de apresentação aos formulários de candidatura de emprego, clientes podem fornecer imagens e exemplos para solicitações de projetos criativos, ou clientes podem enviar documentos e documentação através de formulários de admissão de clientes. Ao incluir campos personalizados de Arquivo nos formulários, partes interessadas externas podem fornecer arquivos e recursos importantes para enriquecer os registros gerados a partir de envios de formulários. Isso oferece o benefício de conteúdo coletado sem exigir que usuários externos tenham contas dentro do Blue.

## Acesso a Arquivos via API

Como qualquer outro arquivo do sistema, arquivos carregados via campo personalizado de Arquivo podem ser acessados e gerenciados através da [API do Blue](/pt/docs/integrations/api). Quando arquivos são anexados a registros usando o campo de Arquivo, esses arquivos ficam disponíveis através das chamadas padrão da API relacionadas a arquivos. Desenvolvedores podem aproveitar todo o conjunto de capacidades da API para baixar, carregar, organizar e integrar arquivos do campo personalizado de Arquivo programaticamente. Seja processando automaticamente recursos carregados ou adicionando metadados, a API permite integração perfeita fornecendo acesso unificado a todos os arquivos adicionados pelo usuário no Blue.

Ao anexar documentos e recursos frequentemente usados, os campos de Arquivo eliminam a busca em múltiplas localizações para encontrar o conteúdo vinculado a um registro. Isso oferece acesso rápido e centralizado aos arquivos que você precisa.
