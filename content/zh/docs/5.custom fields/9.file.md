文件自定义字段允许您直接向 Blue 中的记录上传和附加文件。支持几乎所有文件类型，包括文档、电子表格、图片、PDF 等。

![File 1](/docs/File_1.png "File 1")

结构化的文件上传确保每个人都能看到相关文件

每个文件字段允许上传多个文件，每个文件最大 5GB。上传的文件，如图片、PDF 和 Word 文档，如果文件类型支持，可以进行预览。所有项目用户都可以下载附加的文件以访问相关的记录信息和内容。

![File 2](/docs/File_2.png "File 2")

用户可以在文件自定义字段中预览和下载文件

上传的文件在项目范围内的['文件' 选项卡](/zh/docs/projects/file-management)下共享。但是，具有受限记录访问权限的自定义用户角色将无法看到这些文件。项目文件选项卡支持创建文件夹来组织文件，但无法直接从自定义字段本身将文件分类到这些文件夹中。

::callout

icon: lightbulb

不维护文件版本历史记录——新上传的文件不会替换现有文件。文件也不包含在 CSV 导出报告中。

## 表单集成

文件自定义字段[与表单无缝集成](/zh/docs/projects/forms)，允许外部用户在提交信息时上传文件。例如，候选人可以将简历和求职信附加到求职申请表单中，客户可以为创意项目请求提供图片和示例，或者客户可以通过客户接收表单提交文件和文档。通过在表单中包含文件自定义字段，外部利益相关者可以提供重要的文件和资产来丰富从表单提交生成的记录。这提供了收集内容的好处，而无需外部用户在 Blue 中拥有账户。

## API 文件访问

与任何其他系统文件一样，通过文件自定义字段上传的文件可以通过 [Blue API](/zh/docs/integrations/api) 进行访问和管理。当使用文件字段将文件附加到记录时，这些文件通过标准的与文件相关的 API 调用变得可用。开发人员可以利用完整的 API 功能套件来以编程方式下载、上传、组织和集成来自文件自定义字段的文件。无论是自动处理上传的资产还是附加元数据，API 都通过提供对 Blue 中所有用户添加文件的统一访问来实现无缝集成。

通过附加经常使用的文档和资产，文件字段消除了在多个位置搜索链接到记录的内容的需要。这提供了对您需要的文件的快速、集中访问。
