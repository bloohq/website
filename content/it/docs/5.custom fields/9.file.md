Il campo personalizzato File consente di caricare e allegare file direttamente ai record in Blue. È supportato praticamente qualsiasi tipo di file, inclusi documenti, fogli di calcolo, immagini, PDF e molto altro.

![File 1](/docs/File_1.png "File 1")

I caricamenti di file strutturati garantiscono che tutti abbiano visibilità sui file correlati

Ogni campo file consente di caricare più file, con ciascun file fino a 5GB di dimensione. I file caricati, come immagini, PDF e documenti Word, possono essere visualizzati in anteprima se il tipo di file è supportato. Tutti gli utenti del progetto possono scaricare i file allegati per accedere alle informazioni e al contenuto rilevanti del record.

![File 2](/docs/File_2.png "File 2")

Gli utenti possono visualizzare in anteprima e scaricare i file nel campo personalizzato File

I file caricati sono condivisi a livello di progetto nella [scheda 'Files'](/it/docs/projects/file-management). Tuttavia, i ruoli utente personalizzati con accesso limitato ai record non vedranno quei file. La scheda Files del progetto supporta la creazione di cartelle per organizzare i file, ma i file non possono essere ordinati in queste cartelle direttamente dal campo personalizzato stesso.

::callout
---

icon: lightbulb
---

La cronologia delle versioni dei file non viene mantenuta — i nuovi caricamenti non sostituiscono i file esistenti. I file inoltre non sono inclusi nei report di esportazione CSV.
::

## Integrazione con i Forms

Il campo personalizzato File [si integra perfettamente con i Forms](/it/docs/projects/forms), consentendo agli utenti esterni di caricare file quando inviano informazioni. Ad esempio, i candidati possono allegare CV e lettere di presentazione ai moduli di candidatura, i clienti possono fornire immagini ed esempi per richieste di progetti creativi, o i clienti possono inviare documenti e documentazione attraverso moduli di acquisizione clienti. Includendo campi personalizzati File nei moduli, le parti interessate esterne possono fornire file e risorse importanti per arricchire i record generati dalle invii dei moduli. Questo offre il vantaggio del contenuto raccolto senza richiedere agli utenti esterni di avere account all'interno di Blue.

## Accesso ai File tramite API

Come qualsiasi altro file di sistema, i file caricati tramite il campo personalizzato File possono essere accessibili e gestiti attraverso l'[API di Blue](/it/docs/integrations/api). Quando i file sono allegati ai record utilizzando il campo File, quei file diventano disponibili attraverso le chiamate API standard relative ai file. Gli sviluppatori possono sfruttare l'intera suite di capacità API per scaricare, caricare, organizzare e integrare programmaticamente i file dal campo personalizzato File. Che si tratti di elaborare automaticamente risorse caricate o aggiungere metadati, l'API consente un'integrazione senza soluzione di continuità fornendo accesso unificato a tutti i file aggiunti dagli utenti in Blue.

Allegando documenti e risorse utilizzati frequentemente, i campi File eliminano la ricerca in più posizioni per trovare il contenuto collegato a un record. Questo fornisce accesso rapido e centralizzato ai file di cui avete bisogno.
