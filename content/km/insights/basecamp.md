## ការណែនាំអំពី Basecamp

Basecamp គឺជាអ្នកគ្រប់គ្រងគម្រោងដំបូងៗ — វាត្រូវបានចាប់ផ្តើមនៅឆ្នាំ 2004!

វេទិកានេះបានចាប់ផ្តើមជាឧបករណ៍គ្រប់គ្រងគម្រោងក្នុងស្ថាប័នដែលបង្កើតឡើងដោយក្រុមហ៊ុនរចនាគេហទំព័រ [37signals](https://37signals.com/), ដែលត្រូវបានបង្កើតឡើងដោយ [Jason Fried](https://world.hey.com/jason) និង [David Heinemeier Hansson](https://dhh.dk/). 

David ក៏ជាអ្នកបង្កើត [វេទិកាបើកប្រាក់ Ruby on Rails](https://rubyonrails.org/) ដែលត្រូវបានប្រើប្រាស់ដោយក្រុមហ៊ុនដូចជា [GitHub](https://github.com/), [Shopify](https://www.shopify.com/), [Airbnb](https://www.airbnb.com/), [Square](https://squareup.com/), [Coinbase](https://www.coinbase.com/), និង [Zendesk](https://www.zendesk.com/).

![](/insights/basecamp-home.png)

Basecamp ត្រូវបានអភិវឌ្ឍន៍ដើម្បីដោះស្រាយតម្រូវការរបស់ក្រុមហ៊ុនសម្រាប់ការគ្រប់គ្រងគម្រោង និងការប្រាស្រ័យទាក់ទងជាមួយអតិថិជន។ នេះគឺជារឿងដែល [ស្រដៀងគ្នាជាមួយរឿងរ៉ាវរបស់ ប្លូ!](/about)

បន្ទាប់ពីការចេញផ្សាយ Basecamp បានទទួលការពេញនិយមយ៉ាងឆាប់រហ័ស បណ្តាលឲ្យអ្នកបង្កើតផ្តោតអារម្មណ៍លើវានិងចុងក្រោយបានប្តូរឈ្មោះក្រុមហ៊ុនរបស់ពួកគេជាអ្នកគ្រប់គ្រងគម្រោង Basecamp នៅឆ្នាំ 2014 ដែលគឺជាឆ្នាំទីដប់បន្ទាប់ពីការចេញផ្សាយផលិតផលដំបូង។ ពួកគេផ្តោតលើ *"ការគ្រប់គ្រងគម្រោងដែលមានភាពសាមញ្ញ និងចRefreshing"* ដូចជា [ប្លូ](/)។

ក្នុងរយៈពេលជាច្រើនឆ្នាំ Basecamp បានដំណើរការប្លុកដ៏អស្ចារ្យមួយដែលមានឈ្មោះ [Signal vs Noise](https://signalvnoise.com/) ដែលជាអត្ថបទដែលអាចអានបានយ៉ាងល្អ។

ក្រុម Basecamp ក៏បានបោះពុម្ពសៀវភៅជាច្រើនដែលមានគំនិតខុសគ្នាដែលមានគុណភាពខ្ពស់។

- **[Shape Up](https://basecamp.com/books/shapeup)**: ផ្តល់ឧបករណ៍ដល់ក្រុមដើម្បីដោះស្រាយហានិភ័យ និងអ្វីដែលមិនច្បាស់នៅរៀងរាល់ដំណាក់កាលនៃដំណើរការអភិវឌ្ឍផលិតផល។
- **[It Doesn't Have to Be Crazy at Work](https://basecamp.com/books/calm)**: ធ្វើការពិភាក្សាថា បរិយាកាសការងារដែលមានភាពស្ងប់ស្ងាត់ និងមានការផ្តោតអារម្មណ៍គឺមានផលិតភាព និងអាចរក្សាទុកបានល្អជាងបរិយាកាសដែលគ្រប់គ្រងដោយភាពរំខាន និងសម្ពាធខ្ពស់។
- **[Rework](https://basecamp.com/books/rework)**: ជំរុញការប្រកួតប្រជែងធម្មតា ដោយគាំទ្រការប្រកួតប្រជែងដែលមានភាពសាមញ្ញ និងមានប្រសិទ្ធភាពខ្ពស់។
- **[Remote](https://basecamp.com/books/remote)**: ផ្តល់អត្ថប្រយោជន៍ និងភាពអាចធ្វើបាននៃការងារពីចម្ងាយ។
- **[Getting Real](https://basecamp.com/books/getting-real)**: ផ្តល់អនុសាសន៍អំពីការបង្កើតកម្មវិធីគេហទំព័រ ដែលផ្តោតលើភាពសាមញ្ញ និងប្រសិទ្ធភាព។

## ការណែនាំអំពី ប្លូ
ប្លូ ត្រូវបានបង្កើតឡើងនៅឆ្នាំ 2018 ដោយ [Emanuele "Manny" Faja](https://www.linkedin.com/in/efaja/) ជាមួយបេសកកម្មច្បាស់លាស់៖ 

> ដើម្បីរៀបចំការងារនៅលើពិភពលោកដោយការបង្កើតវេទិកាគ្រប់គ្រងគម្រោងល្អបំផុតនៅលើផែនដី—មានភាពសាមញ្ញ មានអំណាច មានភាពបត់បែន និងមានតម្លៃសមរម្យសម្រាប់គ្រប់គ្នា។

យ៉ាងចម្លែក Manny បានចាប់ផ្តើមការធ្វើការនៅក្នុងពិភពគ្រប់គ្រងគម្រោងដោយប្រើ Basecamp ដើម្បីគ្រប់គ្រង [ក្រុមហ៊ុនរចនាផលិតផលរបស់គាត់](/insights/agency-success-guide)។ គាត់ស្រលាញ់ភាពសាមញ្ញរបស់ Basecamp (អ្នកអាចរកឃើញ [ការពិនិត្យ Basecamp របស់គាត់នៅលើគេហទំព័រ Basecamp!](https://basecamp.com/customers)) ប៉ុន្តែពេលដែលក្រុមហ៊ុនរបស់គាត់ធំឡើង គាត់ត្រូវការអ្វីដែលមានអំណាចច្រើនជាងនេះ ហើយដូច្នេះគាត់បានផ្លាស់ទៅ [Asana](https://asana.com)។ 

ទោះយ៉ាងណា ភាពសាមញ្ញបានបាត់បង់ ហើយវាបានក្លាយជាការលំបាកសម្រាប់ភាគច្រើននៃអតិថិជនរបស់គាត់!

Manny ចង់បានអ្វីដែល *មានភាពងាយស្រួលដូច Basecamp* និង *មានអំណាចដូច Asana*។ 

ដូច Basecamp, ប្លូ មានភាពខ្ពស់ក្នុងការបង្កើតដោយខ្លួនឯង។ យើងជាក្រុមតិចនៅក្នុងឧស្សាហកម្មនេះ ដែលភាគច្រើននៃការប្រកួតបានប្រមូលប្រាក់រាប់រយលានដុល្លារពីការវិនិយោគ។ វិធីសាស្ត្រនេះបានអនុញ្ញាតឱ្យ ប្លូ រក្សាទំនាក់ទំនងជាមួយចក្ខុវិស័យរបស់ខ្លួន និងផ្តោតលើតម្រូវការរបស់អ្នកប្រើប្រាស់ និងគុណភាពផលិតផល ជំនួសការត្រូវតែច្របូកច្របល់នឹងតម្រូវការរបស់អ្នកវិនិយោគជាមួយតម្រូវការរបស់អតិថិជន។

នៅខាងផ្ទុយ ប្រសិនបើអ្នកជឿថាតម្រូវការផ្ទុករបស់អ្នកនឹងសមស្របនៅក្នុងគម្រោង Basecamp ផែនការរបស់ពួកគេអាចដំណើរការសម្រាប់អ្នក។ ការផ្ទុក 500GB នៅក្នុងផែនការមូលដ្ឋានគឺមានភាពសម្បូរបែបសម្រាប់ក្រុមច្រើន ហើយការលោតទៅ 5TB នៅក្នុងផែនការប្រូអត្រាអត្រា ផ្តល់កន្លែងសម្រាប់ការលូតលាស់។ តែសូមប្រាកដថាបានពិនិត្យអំពីកំណត់ការផ្ទុករបស់ Basecamp ប្រសិនបើអ្នកត្រូវការជាមួយឯកសារដែលមានទំហំលើស 5GB ជាញឹកញាប់។

## APIs & Webhooks

### APIs

Basecamp មាន [Rest API ដែលមានឯកសារល្អ](https://github.com/basecamp/api) ប៉ុន្តែ Basecamp ក៏ទទួលស្គាល់ថាវាមិនគ្រប់គ្រាន់ទេ៖

> API របស់ Basecamp មិនមានគោលបំណងដើម្បីជាម៉ូដែលពេញលេញនៃអ្វីដែលអ្នកអាចធ្វើបាននៅក្នុងកម្មវិធីរបស់យើងទាំងអស់។ សេចក្តីពិតគឺថា តិចជាងកន្លះភាគរយនៃអ្នកប្រើប្រាស់ចុងក្រោយរបស់យើងបានអនុវត្តកម្មវិធីដែលបញ្ចូលជាមួយ Basecamp ឬសរសេរ ការបញ្ចូលរបស់ខ្លួន។ ការប្រើប្រាស់ដូចនេះមានន័យថាការកែប្រែ API ដើម្បីរក្សាទំនាក់ទំនងជាមួយលក្ខណៈថ្មីៗគឺមិនមែនជាការប្រកាន់ខ្ជាប់នៅ Basecamp ទេ។ ពេលដែលយើងធ្វើការសម្រាប់អ្នកប្រើប្រាស់ដែលមានតិចជាងកន្លះភាគរយនោះ យើងមិនបានជួយអ្នកប្រើប្រាស់ 99.5% ផ្សេងទៀតទេ។

ការពិតដែលតិចជាងកន្លះភាគរយនៃអ្នកប្រើប្រាស់ចុងក្រោយរបស់ពួកគេបានអនុវត្ត API គឺប្រហែលជាមានមូលហេតុពីព្រោះគោលដៅរបស់ពួកគេគឺជាអាជីវកម្មតូចៗដែលកំពុងស្វែងរកអ្វីដែលមានភាពសាមញ្ញ។ 

[API GraphQL របស់ ប្លូ](/platform/api) មានការក覆គ្រប់លក្ខណៈទាំងអស់នៅក្នុង ប្លូ។ 

ដូច្នេះ *អ្វីដែលអ្នកអាចធ្វើតាមរយៈចំណុចប្រទាក់របស់យើង អ្នកអាចបញ្ជូនដោយកម្មវិធី*។ នេះគឺជាជម្រើសដែលបានរចនាឡើងដោយចេតនា នៅពេលដែលយើងបានបង្កើត API របស់យើងដំបូង ដើម្បីផ្តល់ឱ្យអ្នកអភិវឌ្ឍន៍នូវការចូលដំណើរការពេញលេញទៅលើអ្វីៗទាំងអស់។ 

រាល់ពេលដែលយើង [ចេញផ្សាយលក្ខណៈថ្មី](/platform/changelog) API ត្រូវបានធ្វើអាប់ដេតដើម្បីគាំទ្រលក្ខណៈថ្មីៗទាំងនេះ។ 

### Webhooks

Webhooks គឺជាសារអូតូដែលផ្ញើពីកម្មវិធីមួយទៅកាន់កម្មវិធីមួយទៀតនៅពេលដែលមានព្រឹត្តិការណ៍ជាក់លាក់កើតឡើង។ នៅក្នុងបរិបទនៃកម្មវិធីគ្រប់គ្រងគម្រោង Webhooks មានប្រយោជន៍ព្រោះវាអនុញ្ញាតឱ្យមានការប្រាស្រ័យទាក់ទងដោយគ្មានឈប់ឈរ រវាងឧបករណ៍ផ្សេងៗ។ ឧទាហរណ៍ នៅក្នុងកម្មវិធីគ្រប់គ្រងគម្រោង Webhooks អាចបង្កើតសកម្មភាពដូចជា ការអាប់ដេតស្ថានភាពការងារ ការបង្កើតការជូនដំណឹង ឬការសមកាលកម្មទិន្នន័យជាមួយកម្មវិធីផ្សេងទៀតក្នុងពេលពិត។ វាអនុញ្ញាតឱ្យមានការបញ្ចូលបានល្អរវាងឧបករណ៍គ្រប់គ្រងគម្រោង និងប្រព័ន្ធផ្សេងទៀត ដើម្បីធ្វើឱ្យការងារល្អប្រសើរ និងធ្វើឱ្យមានប្រសិទ្ធភាព។

Basecamp បានមាន [webhooks នៅក្នុង Basecamp 3](https://github.com/basecamp/bc3-api/blob/master/sections/webhooks.md) ប៉ុន្តែគ្មានការបញ្ជាក់អំពី Basecamp 4 ដែលមាន webhooks ហើយយើងមិនអាចរកឃើញអ្វីនៅក្នុងការធ្វើតេស្តកម្មវិធីរបស់យើងទេ។ 

ទោះយ៉ាងណា [Basecamp មានការបញ្ចូលជាមួយកម្មវិធីបញ្ចូលទីបញ្ចូល](https://basecamp.com/extras)។ សម្រាប់ការប្រៀបធៀប [អ្នកក៏អាចមើលបញ្ជីនៃការបញ្ចូលដែល ប្លូ ផ្តល់ឱ្យ](/platform/integrations)។
