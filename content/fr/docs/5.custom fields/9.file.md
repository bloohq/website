Le champ personnalisé Fichier permet de télécharger et d'attacher des fichiers directement aux enregistrements dans Blue. Pratiquement tous les types de fichiers sont pris en charge, y compris les documents, feuilles de calcul, images, PDF et plus encore.

![File 1](/docs/File_1.png "File 1")

Les téléchargements de fichiers structurés garantissent que tout le monde a une visibilité sur les fichiers associés

Chaque champ fichier permet de télécharger plusieurs fichiers, chaque fichier pouvant atteindre 5 Go. Les fichiers téléchargés, tels que les images, PDF et documents Word, peuvent être prévisualisés si le type de fichier est pris en charge. Tous les utilisateurs du projet peuvent télécharger les fichiers attachés pour accéder aux informations et au contenu pertinents de l'enregistrement.

![File 2](/docs/File_2.png "File 2")

Les utilisateurs peuvent prévisualiser et télécharger des fichiers dans le champ personnalisé Fichier

Les fichiers téléchargés sont partagés à l'échelle du projet sous l'['onglet Fichiers'](/fr/docs/projects/file-management). Cependant, les rôles d'utilisateur personnalisés avec un accès restreint aux enregistrements ne verront pas ces fichiers. L'onglet Fichiers du projet prend en charge la création de dossiers pour organiser les fichiers, mais les fichiers ne peuvent pas être triés dans ces dossiers depuis le champ personnalisé lui-même.

::callout
---

icon: lightbulb
---

L'historique des versions de fichiers n'est pas maintenu — les nouveaux téléchargements ne remplacent pas les fichiers existants. Les fichiers ne sont pas non plus inclus dans les rapports d'exportation CSV.
::

## Intégration avec les Formulaires

Le champ personnalisé Fichier [s'intègre parfaitement avec les Formulaires](/fr/docs/projects/forms), permettant aux utilisateurs externes de télécharger des fichiers lors de la soumission d'informations. Par exemple, les candidats peuvent attacher des CV et lettres de motivation aux formulaires de candidature, les clients peuvent fournir des images et exemples pour les demandes de projets créatifs, ou les clients peuvent soumettre des documents et de la documentation via des formulaires d'admission client. En incluant des champs personnalisés Fichier sur les formulaires, les parties prenantes externes peuvent fournir des fichiers et ressources importants pour enrichir les enregistrements générés à partir des soumissions de formulaires. Cela offre l'avantage du contenu collecté sans nécessiter que les utilisateurs externes aient des comptes dans Blue.

## Accès aux Fichiers via l'API

Comme tout autre fichier système, les fichiers téléchargés via le champ personnalisé Fichier peuvent être accédés et gérés via l'[API Blue](/fr/docs/integrations/api). Lorsque des fichiers sont attachés aux enregistrements en utilisant le champ Fichier, ces fichiers deviennent disponibles via les appels API standard liés aux fichiers. Les développeurs peuvent exploiter l'ensemble complet des capacités de l'API pour télécharger, téléverser, organiser et intégrer programmatiquement les fichiers du champ personnalisé Fichier. Que ce soit pour traiter automatiquement les ressources téléchargées ou ajouter des métadonnées, l'API permet une intégration transparente en fournissant un accès unifié à tous les fichiers ajoutés par les utilisateurs dans Blue.

En attachant les documents et ressources fréquemment utilisés, les champs Fichier éliminent la recherche dans plusieurs emplacements pour trouver le contenu lié à un enregistrement. Cela offre un accès rapide et centralisé aux fichiers dont vous avez besoin.
