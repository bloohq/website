Les actions sont les opérations que Blue exécute automatiquement lorsqu'une automatisation est déclenchée. Une fois qu'une condition de déclenchement est remplie, les actions exécutent des tâches spécifiques pour rationaliser votre flux de travail.

## Actions disponibles

Blue fournit un ensemble complet d'actions qui peuvent être combinées pour créer des automatisations puissantes :

### Actions de gestion des tâches
- **Changer de liste de tâches** : Déplacer un enregistrement vers une liste différente dans le projet
- **Marquer comme terminé** : Marquer automatiquement un enregistrement comme fait
- **Marquer comme incomplet** : Remettre un enregistrement au statut incomplet
- **Copier la tâche** : Créer un duplicata de l'enregistrement avec les options spécifiées

### Actions d'attribution
- **Ajouter un assigné** : Attribuer un ou plusieurs membres de l'équipe à un enregistrement
- **Supprimer un assigné** : Retirer des assignés spécifiques d'un enregistrement

### Actions de gestion des dates
- **Changer la date d'échéance** : Définir ou mettre à jour la date d'échéance d'un enregistrement
- **Supprimer la date d'échéance** : Effacer la date d'échéance d'un enregistrement

### Actions d'organisation
- **Ajouter une étiquette** : Appliquer des étiquettes pour catégoriser et organiser les enregistrements
- **Supprimer une étiquette** : Retirer des étiquettes spécifiques des enregistrements
- **Ajouter une couleur** : Appliquer un codage couleur aux enregistrements pour l'organisation visuelle
- **Supprimer une couleur** : Retirer le codage couleur des enregistrements

### Actions de liste de contrôle
- **Marquer un élément de liste de contrôle comme fait** : Terminer des éléments spécifiques de la liste de contrôle
- **Marquer un élément de liste de contrôle comme non fait** : Remettre les éléments de la liste de contrôle à incomplet
- **Créer une liste de contrôle** : Ajouter de nouvelles listes de contrôle avec des éléments aux enregistrements

### Actions de champs personnalisés
- **Ajouter un champ personnalisé** : Définir des valeurs pour les champs personnalisés sur les enregistrements
- **Supprimer un champ personnalisé** : Effacer les valeurs des champs personnalisés

### Actions de communication
- **[Envoyer un email](/fr/docs/automations/actions/email-automations)** : Envoyer des emails automatisés avec les données d'enregistrement
- **Générer un PDF** : Créer des documents PDF à partir de modèles de documents portables
- **Faire une requête HTTP** : Intégrer avec des services externes via des webhooks

## Comment fonctionnent les actions

Les actions peuvent être enchaînées pour créer des flux de travail complexes. Par exemple, lorsqu'un enregistrement est marqué comme terminé, vous pourriez :
1. Le déplacer vers une liste "Terminé"
2. Envoyer un email de notification au client
3. Créer une tâche de suivi
4. Mettre à jour un champ personnalisé avec la date de finalisation

## Configuration des actions

Chaque type d'action a des options de configuration spécifiques :
- **Sélection de cible** : Choisir quels enregistrements ou utilisateurs sont affectés
- **Logique conditionnelle** : Définir des conditions pour quand les actions doivent s'exécuter
- **Mappage de données** : Utiliser des balises de fusion pour inclure des données dynamiques des enregistrements
- **Timing** : Certaines actions supportent les délais ou l'exécution programmée

## Bonnes pratiques

1. **Testez d'abord** : Testez toujours les automatisations avec des données d'exemple avant de les activer
2. **Commencez simple** : Commencez par des actions uniques et ajoutez la complexité progressivement
3. **Documentez l'objectif** : Utilisez une dénomination claire pour décrire ce que fait chaque automatisation
4. **Surveillez les résultats** : Examinez les journaux d'automatisation pour assurer le comportement attendu
5. **Évitez les conflits** : Attention à ne pas créer d'actions qui s'annulent mutuellement

## Permissions

Seuls les Administrateurs de projet peuvent créer et modifier les actions d'automatisation. Tous les membres de l'équipe peuvent voir les automatisations actives pour comprendre le flux de travail du projet.
