---
title: Déclencheurs
description: 
---

Un déclencheur est un événement qui active une automation. Par exemple, un déclencheur peut être l'achèvement d'un enregistrement, un changement dans l'assigné d'un enregistrement, ou une modification dans un champ personnalisé. Lorsque cette condition prédéfinie est remplie, l'automation est activée.

Voici tous les déclencheurs d'automation possibles dans Blue :

- **Un Nouvel Enregistrement est Créé** : Se déclenche lorsqu'un nouvel enregistrement est ajouté au projet. Utile pour initier des actions de suivi ou des notifications liées aux nouvelles tâches ou entrées. Ceci se déclenche également lorsqu'il y a une [nouvelle soumission de formulaire](/fr/docs/projects/forms).
- **Un Enregistrement est Déplacé vers une Autre Liste** : S'active lorsqu'un enregistrement est transféré d'une liste à une autre au sein du projet, indiquant un changement dans son étape ou statut.
- **Un Enregistrement est Marqué comme Terminé** : Se déclenche lorsque le statut d'un enregistrement est mis à jour vers 'terminé', signalant la fin d'une tâche ou d'un processus.
- **Un Enregistrement est Marqué comme Non Terminé** : S'active lorsqu'un enregistrement précédemment marqué comme terminé est remis à un statut non terminé.
- **Quelqu'un est Assigné à un Enregistrement** : Se déclenche lorsqu'un nouvel assigné est ajouté à un enregistrement, indiquant sa responsabilité pour cette tâche ou élément.
- **Quelqu'un est Désassigné d'un Enregistrement** : S'active lorsqu'un assigné est retiré d'un enregistrement, signifiant un changement dans la responsabilité de la tâche.
- **La Date d'Échéance Change** : Se déclenche lorsque la date d'échéance d'un enregistrement est modifiée, ce qui peut être critique pour les tâches sensibles au temps.
- **La Date d'Échéance est Supprimée** : S'active lorsque la date d'échéance d'un enregistrement est effacée, indiquant possiblement un changement dans les priorités ou les délais.
- **La Date d'Échéance a Expiré** : Se déclenche lorsque la date d'échéance d'un enregistrement passe sans achèvement, mettant en évidence des retards potentiels ou des oublis. C'est une excellente automation pour les managers, car ils peuvent se signaler des enregistrements/tâches s'ils deviennent en retard.
- **Une Étiquette est Ajoutée à un Enregistrement** : S'active lorsqu'une nouvelle étiquette est ajoutée à un enregistrement, utile pour la catégorisation et le suivi des changements dans les attributs des tâches.
- **Une Étiquette est Supprimée d'un Enregistrement** : Se déclenche lorsqu'une étiquette est supprimée d'un enregistrement, indiquant un changement dans sa catégorisation ou ses propriétés.
- **Toutes les Listes de Contrôle sont Marquées comme Terminées** : S'active lorsque chaque élément dans la liste de contrôle d'un enregistrement est coché, indiquant l'achèvement de toutes les sous-tâches.
- **Une Liste de Contrôle est Marquée comme Non Terminée** : Se déclenche lorsqu'un élément dans la liste de contrôle d'un enregistrement est marqué non terminé, signalant des sous-tâches inachevées.
- **Un Enregistrement est Déplacé/Copié depuis un Autre Projet** : S'active lorsqu'un enregistrement est apporté dans le projet actuel depuis un projet différent, utile pour suivre les flux de travail inter-projets. Le déclencheur s'applique que l'enregistrement soit manuellement déplacé ou copié par un utilisateur, ou automatiquement transféré par une autre automation, fournissant un moyen complet de suivre les activités inter-projets.
- **Un Champ Personnalisé est Ajouté à un Enregistrement** : Se déclenche lorsque des données sont écrites dans un champ personnalisé spécifique d'un enregistrement, indiquant un changement ou ajout d'informations spécifiques. Ceci ne fonctionne qu'avec les cases à cocher, les champs personnalisés de sélection simple et multiple.
- **Un Champ Personnalisé est Supprimé d'un Enregistrement** : S'active lorsque des données sont supprimées d'un champ personnalisé spécifique d'un enregistrement, signifiant une modification dans la structure d'informations de l'enregistrement.
- **Une Couleur est ajoutée à un Enregistrement** : Se déclenche lorsqu'une couleur prédéfinie est assignée à un enregistrement, indiquant la priorité, le progrès, ou la catégorisation. Par exemple, ajouter une couleur rouge pour les tâches haute priorité ou une couleur verte pour les éléments en cours.
- **Une Couleur est supprimée d'un Enregistrement** : Se déclenche lorsqu'une couleur prédéfinie est supprimée d'un enregistrement, signalant un changement de statut ou de priorité. Par exemple, supprimer une couleur rouge d'une tâche lorsqu'elle n'est plus haute priorité ou supprimer une couleur verte lorsqu'une tâche est terminée.
- **Bouton de Champ Personnalisé Cliqué** : Se déclenche lorsqu'un utilisateur clique sur un bouton de champ personnalisé sur un enregistrement. Ceci permet des flux de travail interactifs où les utilisateurs peuvent déclencher manuellement des automations en cliquant sur des boutons désignés dans les enregistrements.
