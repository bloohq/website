El campo personalizado de Archivo permite subir y adjuntar archivos directamente a los registros en Blue. Se admite prácticamente cualquier tipo de archivo, incluyendo documentos, hojas de cálculo, imágenes, PDFs y más.

![File 1](/docs/File_1.png "File 1")

Las subidas de archivos estructuradas aseguran que todos tengan visibilidad sobre los archivos relacionados

Cada campo de archivo permite subir múltiples archivos, con cada archivo de hasta 5GB de tamaño. Los archivos subidos, como imágenes, PDFs y documentos de Word, pueden previsualizarse si el tipo de archivo es compatible. Todos los usuarios del proyecto pueden descargar los archivos adjuntos para acceder a la información y contenido relevante del registro.

![File 2](/docs/File_2.png "File 2")

Los usuarios pueden previsualizar y descargar archivos en el campo personalizado de Archivo

Los archivos subidos se comparten a nivel de proyecto bajo la [pestaña 'Archivos'](/es/docs/projects/file-management). Sin embargo, los roles de usuario personalizados con acceso restringido a registros no verán esos archivos. La pestaña de Archivos del proyecto admite la creación de carpetas para organizar archivos, pero los archivos no pueden ordenarse en estas carpetas desde el campo personalizado mismo.

::callout
---

icon: lightbulb

No se mantiene el historial de versiones de archivos — las nuevas subidas no reemplazan los archivos existentes. Los archivos tampoco se incluyen en los reportes de exportación CSV.
::

## Integración con Formularios

El campo personalizado de Archivo [se integra perfectamente con Formularios](/es/docs/projects/forms), permitiendo a usuarios externos subir archivos al enviar información. Por ejemplo, los candidatos pueden adjuntar CVs y cartas de presentación a formularios de solicitud de empleo, los clientes pueden proporcionar imágenes y ejemplos para solicitudes de proyectos creativos, o los clientes pueden enviar documentación y papeleo a través de formularios de registro de clientes. Al incluir campos personalizados de Archivo en formularios, las partes interesadas externas pueden proporcionar archivos y recursos importantes para enriquecer los registros generados a partir de envíos de formularios. Esto ofrece el beneficio del contenido recolectado sin requerir que los usuarios externos tengan cuentas dentro de Blue.

## Acceso a Archivos por API

Como cualquier otro archivo del sistema, los archivos subidos a través del campo personalizado de Archivo pueden accederse y gestionarse a través de la [API de Blue](/es/docs/integrations/api). Cuando los archivos se adjuntan a registros usando el campo de Archivo, esos archivos se vuelven disponibles a través de las llamadas estándar de API relacionadas con archivos. Los desarrolladores pueden aprovechar el conjunto completo de capacidades de API para descargar, subir, organizar e integrar archivos del campo personalizado de Archivo de manera programática. Ya sea procesando automáticamente recursos subidos o agregando metadatos, la API permite una integración perfecta al proporcionar acceso unificado a todos los archivos agregados por usuarios en Blue.

Al adjuntar documentos y recursos de uso frecuente, los campos de Archivo eliminan la búsqueda a través de múltiples ubicaciones para encontrar el contenido vinculado a un registro. Esto ofrece acceso rápido y centralizado a los archivos que usted necesita.
