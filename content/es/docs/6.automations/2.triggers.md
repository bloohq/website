---
title: Disparadores
description: 
---

Un disparador es un evento que activa una automatización. Por ejemplo, un disparador podría ser la finalización de un registro, un cambio en el asignado de un registro, o un cambio en un campo personalizado. Cuando se cumple esta condición predefinida, la automatización se activa.

Estos son todos los disparadores de automatización posibles en Blue:

- **Se Crea un Nuevo Registro**: Se activa cuando se agrega un nuevo registro al proyecto. Útil para iniciar acciones de seguimiento o notificaciones relacionadas con nuevas tareas o entradas. Esto también se activa cuando hay un [nuevo envío de formulario](/es/docs/projects/forms).
- **Un Registro se Mueve a Otra Lista**: Se activa cuando un registro se transfiere de una lista a otra dentro del proyecto, indicando un cambio en su etapa o estado.
- **Un Registro se Marca como Completo**: Se activa cuando el estado de un registro se actualiza a 'completo', señalando el final de una tarea o proceso.
- **Un Registro se Marca como Incompleto**: Se activa cuando un registro previamente marcado como completo se revierte a un estado incompleto.
- **Alguien es Asignado a un Registro**: Se activa cuando se agrega un nuevo asignado a un registro, indicando su responsabilidad por esa tarea o elemento.
- **Alguien es Desasignado de un Registro**: Se activa cuando se remueve un asignado de un registro, significando un cambio en la responsabilidad de la tarea.
- **Cambia la Fecha de Vencimiento**: Se activa cuando se altera la fecha de vencimiento de un registro, lo cual puede ser crítico para tareas sensibles al tiempo.
- **Se Remueve la Fecha de Vencimiento**: Se activa cuando se borra la fecha de vencimiento de un registro, posiblemente indicando un cambio en prioridades o cronogramas.
- **La Fecha de Vencimiento ha Expirado**: Se activa cuando la fecha de vencimiento de un registro pasa sin completarse, resaltando posibles retrasos o descuidos. Esta es una automatización excelente para gerentes, ya que pueden marcarse registros/tareas a sí mismos si se vuelven vencidos.
- **Se Agrega una Etiqueta al Registro**: Se activa cuando se agrega una nueva etiqueta a un registro, útil para categorización y seguimiento de cambios en atributos de tareas.
- **Se Remueve una Etiqueta del Registro**: Se activa cuando se remueve una etiqueta de un registro, indicando un cambio en su categorización o propiedades.
- **Todas las Listas de Verificación se Marcan como Completas**: Se activa cuando cada elemento en la lista de verificación de un registro está marcado, indicando la finalización de todas las sub-tareas.
- **Cualquier Lista de Verificación se Marca como Incompleta**: Se activa cuando cualquier elemento en la lista de verificación de un registro se marca como incompleto, señalando sub-tareas sin terminar.
- **Un Registro se Mueve/Copia desde Otro Proyecto**: Se activa cuando un registro se trae al proyecto actual desde uno diferente, útil para rastrear flujos de trabajo entre proyectos. El disparador aplica ya sea que el registro se mueva o copie manualmente por un usuario, o se transfiera automáticamente a través de otra automatización, proporcionando una forma integral de rastrear actividades entre proyectos.
- **Se Agrega un Campo Personalizado a un Registro**: Se activa cuando se escriben datos en un campo personalizado específico en un registro, indicando un cambio o adición de información específica. Esto solo funciona con casillas de verificación, campos personalizados de selección simple y múltiple.
- **Se Remueve un Campo Personalizado de un Registro**: Se activa cuando se remueven datos de un campo personalizado específico en un registro, significando una modificación en la estructura de información del registro.
- **Se Agrega un Color a un Registro**: Se activa cuando se asigna un color predefinido a un registro, indicando prioridad, progreso, o categorización. Por ejemplo, agregar un color rojo para tareas de alta prioridad o un color verde para elementos en progreso.
- **Se Remueve un Color de un Registro**: Se activa cuando se remueve un color predefinido de un registro, señalando un cambio en estado o prioridad. Por ejemplo, remover un color rojo de una tarea cuando ya no es de alta prioridad o remover un color verde cuando una tarea se completa.
- **Se Hace Clic en Botón de Campo Personalizado**: Se activa cuando un usuario hace clic en un campo personalizado de botón en un registro. Esto habilita flujos de trabajo interactivos donde los usuarios pueden activar manualmente automatizaciones haciendo clic en botones designados dentro de los registros.
