파일 사용자 정의 필드는 Blue의 레코드에 직접 파일을 업로드하고 첨부할 수 있는 기능을 제공합니다. 문서, 스프레드시트, 이미지, PDF 등 거의 모든 파일 형식이 지원됩니다.

![File 1](/docs/File_1.png "File 1")

구조화된 파일 업로드를 통해 모든 사용자가 관련 파일을 확인할 수 있습니다

각 파일 필드는 최대 5GB 크기의 여러 파일을 업로드할 수 있습니다. 이미지, PDF, Word 문서와 같은 업로드된 파일은 파일 형식이 지원되는 경우 미리 볼 수 있습니다. 모든 프로젝트 사용자는 첨부된 파일을 다운로드하여 관련 레코드 정보와 콘텐츠에 액세스할 수 있습니다.

![File 2](/docs/File_2.png "File 2")

사용자는 파일 사용자 정의 필드에서 파일을 미리 보고 다운로드할 수 있습니다

업로드된 파일은 ['파일' 탭](/ko/docs/projects/file-management) 하에서 프로젝트 전체에 공유됩니다. 하지만 레코드 액세스가 제한된 사용자 정의 사용자 역할은 해당 파일을 볼 수 없습니다. 프로젝트 파일 탭은 파일을 정리하기 위한 폴더 생성을 지원하지만, 사용자 정의 필드 내에서 직접 이러한 폴더로 파일을 분류할 수는 없습니다.

::callout
---

icon: lightbulb
---

파일 버전 기록은 유지되지 않습니다 — 새로운 업로드가 기존 파일을 대체하지 않습니다. 또한 파일은 CSV 내보내기 보고서에 포함되지 않습니다.
::

## 양식 통합

파일 사용자 정의 필드는 [양식과 완벽하게 통합](/ko/docs/projects/forms)되어 외부 사용자가 정보를 제출할 때 파일을 업로드할 수 있게 합니다. 예를 들어, 후보자는 채용 지원서에 이력서와 자기소개서를 첨부할 수 있고, 고객은 창작 프로젝트 요청을 위한 이미지와 예시를 제공할 수 있으며, 클라이언트는 고객 접수 양식을 통해 서류와 문서를 제출할 수 있습니다. 양식에 파일 사용자 정의 필드를 포함함으로써 외부 관계자들은 양식 제출로 생성된 레코드를 풍부하게 만드는 중요한 파일과 자산을 제공할 수 있습니다. 이는 외부 사용자가 Blue 내에서 계정을 가질 필요 없이 수집된 콘텐츠의 이점을 제공합니다.

## API 파일 액세스

다른 시스템 파일과 마찬가지로 파일 사용자 정의 필드를 통해 업로드된 파일은 [Blue API](/ko/docs/integrations/api)를 통해 액세스하고 관리할 수 있습니다. 파일 필드를 사용하여 레코드에 파일이 첨부되면 해당 파일은 표준 파일 관련 API 호출을 통해 사용할 수 있게 됩니다. 개발자는 파일 사용자 정의 필드의 파일을 프로그래밍 방식으로 다운로드, 업로드, 정리 및 통합하기 위해 전체 API 기능 세트를 활용할 수 있습니다. 업로드된 자산을 자동으로 처리하거나 메타데이터를 추가하든, API는 Blue의 모든 사용자 추가 파일에 대한 통합된 액세스를 제공하여 원활한 통합을 가능하게 합니다.

자주 사용하는 문서와 자산을 첨부함으로써 파일 필드는 레코드와 연결된 콘텐츠를 찾기 위해 여러 위치를 검색할 필요를 없애줍니다. 이를 통해 필요한 파일에 빠르고 중앙 집중식으로 액세스할 수 있습니다.
