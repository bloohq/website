액션은 자동화가 트리거될 때 Blue가 자동으로 수행하는 작업입니다. 트리거 조건이 충족되면 액션이 특정 작업을 실행하여 워크플로우를 간소화합니다.

## 사용 가능한 액션

Blue는 강력한 자동화를 생성하기 위해 결합할 수 있는 포괄적인 액션 세트를 제공합니다:

### 작업 관리 액션
- **할 일 목록 변경**: 프로젝트 내의 다른 목록으로 레코드 이동
- **완료로 표시**: 레코드를 자동으로 완료로 표시
- **미완료로 표시**: 레코드를 미완료 상태로 되돌리기
- **할 일 복사**: 지정된 옵션으로 레코드의 복제본 생성

### 할당 액션
- **담당자 추가**: 레코드에 한 명 이상의 팀 멤버 할당
- **담당자 제거**: 레코드에서 특정 담당자 제거

### 날짜 관리 액션
- **마감일 변경**: 레코드의 마감일 설정 또는 업데이트
- **마감일 제거**: 레코드에서 마감일 삭제

### 조직 액션
- **태그 추가**: 레코드를 분류하고 정리하기 위한 태그 적용
- **태그 제거**: 레코드에서 특정 태그 제거
- **색상 추가**: 시각적 정리를 위해 레코드에 색상 코딩 적용
- **색상 제거**: 레코드에서 색상 코딩 제거

### 체크리스트 액션
- **체크리스트 항목을 완료로 표시**: 특정 체크리스트 항목 완료
- **체크리스트 항목을 미완료로 표시**: 체크리스트 항목을 미완료로 되돌리기
- **체크리스트 생성**: 레코드에 항목이 포함된 새 체크리스트 추가

### 사용자 정의 필드 액션
- **사용자 정의 필드 추가**: 레코드의 사용자 정의 필드에 값 설정
- **사용자 정의 필드 제거**: 사용자 정의 필드 값 삭제

### 커뮤니케이션 액션
- **[이메일 전송](/ko/docs/automations/actions/email-automations)**: 레코드 데이터와 함께 자동화된 이메일 전송
- **PDF 생성**: 휴대용 문서 템플릿에서 PDF 문서 생성
- **HTTP 요청 만들기**: 웹훅을 통해 외부 서비스와 통합

## 액션 작동 방식

액션은 연결되어 복잡한 워크플로우를 생성할 수 있습니다. 예를 들어, 레코드가 완료로 표시되면 다음과 같은 작업을 수행할 수 있습니다:
1. "완료됨" 목록으로 이동
2. 클라이언트에게 알림 이메일 전송
3. 후속 작업 생성
4. 완료 날짜로 사용자 정의 필드 업데이트

## 액션 구성

각 액션 유형에는 특정 구성 옵션이 있습니다:
- **대상 선택**: 영향을 받을 레코드 또는 사용자 선택
- **조건부 로직**: 액션이 실행될 조건 설정
- **데이터 매핑**: 병합 태그를 사용하여 레코드의 동적 데이터 포함
- **타이밍**: 일부 액션은 지연 또는 예약된 실행 지원

## 모범 사례

1. **먼저 테스트**: 활성화하기 전에 항상 샘플 데이터로 자동화를 테스트하십시오
2. **간단하게 시작**: 단일 액션으로 시작하여 점진적으로 복잡성을 추가하십시오
3. **목적 문서화**: 각 자동화가 수행하는 작업을 설명하는 명확한 이름을 사용하십시오
4. **결과 모니터링**: 예상된 동작을 보장하기 위해 자동화 로그를 검토하십시오
5. **충돌 방지**: 서로를 무효화하는 액션을 생성하지 않도록 주의하십시오

## 권한

프로젝트 관리자만이 자동화 액션을 생성하고 수정할 수 있습니다. 모든 팀 멤버는 프로젝트의 워크플로우를 이해하기 위해 활성 자동화를 볼 수 있습니다.
