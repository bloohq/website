---
title: 트리거
description: 
---

트리거는 자동화를 실행시키는 이벤트입니다. 예를 들어, 트리거는 레코드의 완료, 레코드 담당자의 변경, 또는 사용자 정의 필드의 변경일 수 있습니다. 이러한 미리 정의된 조건이 충족되면 자동화가 활성화됩니다.

다음은 Blue에서 가능한 모든 자동화 트리거입니다:

- **신규 레코드 생성**: 프로젝트에 새로운 레코드가 추가될 때 트리거됩니다. 새로운 작업이나 항목과 관련된 후속 작업이나 알림을 시작하는 데 유용합니다. [새로운 양식 제출](/ko/docs/projects/forms)이 있을 때도 트리거됩니다.
- **레코드가 다른 목록으로 이동**: 레코드가 프로젝트 내에서 한 목록에서 다른 목록으로 이동할 때 활성화되며, 단계나 상태의 변경을 나타냅니다.
- **레코드가 완료로 표시**: 레코드의 상태가 '완료'로 업데이트될 때 트리거되며, 작업이나 프로세스의 종료를 나타냅니다.
- **레코드가 미완료로 표시**: 이전에 완료로 표시된 레코드가 미완료 상태로 되돌려질 때 활성화됩니다.
- **레코드에 담당자 지정**: 레코드에 새로운 담당자가 추가될 때 트리거되며, 해당 작업이나 항목에 대한 책임을 나타냅니다.
- **레코드에서 담당자 해제**: 레코드에서 담당자가 제거될 때 활성화되며, 작업 책임의 변경을 나타냅니다.
- **마감일 변경**: 레코드의 마감일이 변경될 때 트리거되며, 시간에 민감한 작업에서 중요할 수 있습니다.
- **마감일 제거**: 레코드의 마감일이 삭제될 때 활성화되며, 우선순위나 일정의 변경을 나타낼 수 있습니다.
- **마감일 만료**: 레코드의 마감일이 완료 없이 지나갈 때 트리거되며, 잠재적인 지연이나 누락을 강조합니다. 이는 관리자에게 훌륭한 자동화로, 작업이 지연될 때 레코드/작업을 자신에게 플래그할 수 있습니다.
- **레코드에 태그 추가**: 레코드에 새로운 태그가 추가될 때 활성화되며, 작업 속성의 분류 및 변경 추적에 유용합니다.
- **레코드에서 태그 제거**: 레코드에서 태그가 제거될 때 트리거되며, 분류나 속성의 변경을 나타냅니다.
- **모든 체크리스트 완료 표시**: 레코드의 체크리스트에 있는 모든 항목이 체크될 때 활성화되며, 모든 하위 작업의 완료를 나타냅니다.
- **체크리스트 항목 미완료 표시**: 레코드의 체크리스트에 있는 항목이 미완료로 표시될 때 트리거되며, 완료되지 않은 하위 작업을 나타냅니다.
- **레코드가 다른 프로젝트에서 이동/복사**: 레코드가 다른 프로젝트에서 현재 프로젝트로 가져올 때 활성화되며, 프로젝트 간 워크플로우 추적에 도움이 됩니다. 이 트리거는 사용자가 수동으로 레코드를 이동하거나 복사할 때와 다른 자동화를 통해 자동으로 전송될 때 모두 적용되어, 프로젝트 간 활동을 추적하는 포괄적인 방법을 제공합니다.
- **레코드에 사용자 정의 필드 추가**: 레코드의 특정 사용자 정의 필드에 데이터가 입력될 때 트리거되며, 특정 정보의 변경이나 추가를 나타냅니다. 이는 체크박스, 단일 선택 및 다중 선택 사용자 정의 필드에서만 작동합니다.
- **레코드에서 사용자 정의 필드 제거**: 레코드의 특정 사용자 정의 필드에서 데이터가 제거될 때 활성화되며, 레코드의 정보 구조 수정을 나타냅니다.
- **레코드에 색상 추가**: 레코드에 미리 정의된 색상이 할당될 때 트리거되며, 우선순위, 진행 상황 또는 분류를 나타냅니다. 예를 들어, 높은 우선순위 작업에 빨간색을 추가하거나 진행 중인 항목에 녹색을 추가하는 경우입니다.
- **레코드에서 색상 제거**: 레코드에서 미리 정의된 색상이 제거될 때 트리거되며, 상태나 우선순위의 변경을 나타냅니다. 예를 들어, 작업이 더 이상 높은 우선순위가 아닐 때 빨간색을 제거하거나 작업이 완료될 때 녹색을 제거하는 경우입니다.
- **사용자 정의 필드 버튼 클릭**: 사용자가 레코드의 버튼 사용자 정의 필드를 클릭할 때 트리거됩니다. 이를 통해 사용자가 레코드 내의 지정된 버튼을 클릭하여 수동으로 자동화를 트리거할 수 있는 대화형 워크플로우가 가능합니다.
