Автоматизация электронной почты позволяет автоматически отправлять письма при возникновении определенных триггеров, используя данные, хранящиеся в ваших записях Blue. Например, отправка подтверждающего письма при создании новой записи или уведомление менеджера о просроченной задаче.

## Настройка

Начать работу с автоматизацией электронной почты несложно. При первом создании автоматизации мы рекомендуем отправить тестовое письмо себе или коллеге. Это позволит убедиться, что автоматизация функционирует, прежде чем запускать её в работу.

Чтобы создать автоматизацию электронной почты:

- Установите триггер, например:
  - Создается новая запись
  - К записи добавляется тег
  - Запись перемещается в другой список

![EmailAutomation 1](/docs/automation/actions/EmailAutomation_1.png "EmailAutomation 1")

После настройки условия "когда что-то происходит", выберите действие "Отправить письмо"

![EmailAutomation 2](/docs/automation/actions/EmailAutomation_2.png "EmailAutomation 2")

Теперь пришло время настроить детали письма:

![EmailAutomation 3](/docs/automation/actions/EmailAutomation_3.png "EmailAutomation 3")

- Имя отправителя и адрес для ответа, чтобы получатели могли отвечать.
- Адрес получателя (может быть статическим или динамически извлекаться из [пользовательского поля Email](/ru/docs/custom-fields/introduction/email))
- Введите свой собственный адрес электронной почты или тестовый адрес в поле получателя. Если вы хотите включить других, вы также можете добавить адреса CC или BCC.

![EmailAutomation 4](/docs/automation/actions/EmailAutomation_4.png "EmailAutomation 4")

Далее настройте содержимое письма, используя теги слияния:

- Тема письма и содержимое с использованием пользовательских полей для динамического контента с помощью тегов слияния.
- Кликните внутри тела письма, затем вставьте теги слияния, ссылающиеся на имена записей, пользовательские поля и т.д., используя синтаксис {фигурных скобок}.
- Вставьте файловые вложения перетаскиванием или используя значок вложения. Файлы из пользовательских полей типа "Файл" могут автоматически прикрепляться, если они менее 10 МБ.

![EmailAutomation 5](/docs/automation/actions/EmailAutomation_5.png "EmailAutomation 5")

После завершения настройки шаблона подтвердите создание автоматизации. При срабатывании триггера теперь будут отправляться персонализированные письма с использованием данных ваших записей.

## Результат автоматизации

## Примеры использования

**Отправка подтверждающего письма при подаче заявки клиентом через форму приема**

**_Триггер автоматизации_**

Когда новая запись создается через формы -> Отправить письмо

Форма должна также содержать поле для электронной почты, чтобы автоматизированное письмо отправляло правильный ответ клиенту.

**Уведомление исполнителя при создании новой высокоприоритетной задачи**

**_Триггер автоматизации_**

Когда к записи добавляется тег "Приоритет" -> Отправить письмо

В шаблоне письма получателем должен быть {Assignee}, чтобы автоматизированное письмо отправлялось назначенному пользователю.

**Отправка опроса клиенту после закрытия тикета поддержки**

**_Триггер автоматизации_**

Когда запись помечается как завершенная в списке 'Выполнено' -> Отправить письмо

Включите адрес электронной почты клиента в пользовательское поле с подробным описанием проблемы, чтобы вы могли настроить шаблоны с заданными пользовательскими полями, позволяя клиенту получить подробное объяснение проблемы и способа её решения.

**Автоматизация программы найма через отправку подтверждающих писем**

**_Триггер автоматизации_**

Когда заявка подается через формы -> Отправить письмо

Включите адрес электронной почты заявителя в формы, чтобы автоматически захватывать эти данные и отправлять благодарственное письмо при создании записи в списке 'Получено'.

## Основные преимущества

- **Персонализированная коммуникация**: Письма могут использовать данные пользовательских полей из записей с помощью тегов слияния, обеспечивая динамические, индивидуальные сообщения.
- **Автоматические уведомления**: Исключает ручную работу по отправке повторяющихся обновлений и напоминаний по электронной почте.
- **Структурированные рабочие процессы на основе данных**: Автоматизированные письма могут продвигать проекты вперед, запуская действия на основе данных записей.
