Действия — это операции, которые Blue выполняет автоматически при срабатывании автоматизации. После выполнения условия триггера действия выполняют определенные задачи для оптимизации вашего рабочего процесса.

## Доступные действия

Blue предоставляет полный набор действий, которые можно комбинировать для создания мощных автоматизаций:

### Действия управления задачами
- **Изменить список задач**: Переместить запись в другой список в рамках проекта
- **Отметить как выполненное**: Автоматически отметить запись как завершенную
- **Отметить как невыполненное**: Вернуть запись в статус невыполненной
- **Копировать задачу**: Создать дубликат записи с указанными параметрами

### Действия назначения
- **Добавить исполнителя**: Назначить одного или нескольких участников команды на запись
- **Удалить исполнителя**: Удалить определенных исполнителей из записи

### Действия управления датами
- **Изменить срок выполнения**: Установить или обновить срок выполнения записи
- **Удалить срок выполнения**: Очистить срок выполнения записи

### Действия организации
- **Добавить тег**: Применить теги для категоризации и организации записей
- **Удалить тег**: Удалить определенные теги из записей
- **Добавить цвет**: Применить цветовое кодирование к записям для визуальной организации
- **Удалить цвет**: Удалить цветовое кодирование из записей

### Действия чек-листа
- **Отметить пункт чек-листа как выполненный**: Завершить определенные пункты чек-листа
- **Отметить пункт чек-листа как невыполненный**: Вернуть пункты чек-листа в статус невыполненных
- **Создать чек-лист**: Добавить новые чек-листы с пунктами к записям

### Действия пользовательских полей
- **Добавить пользовательское поле**: Установить значения для пользовательских полей в записях
- **Удалить пользовательское поле**: Очистить значения пользовательских полей

### Действия коммуникации
- **[Отправить email](/ru/docs/automations/actions/email-automations)**: Отправить автоматические электронные письма с данными записи
- **Создать PDF**: Создать PDF-документы из портативных шаблонов документов
- **Выполнить HTTP-запрос**: Интегрироваться с внешними сервисами через webhooks

## Как работают действия

Действия можно объединять в цепочки для создания сложных рабочих процессов. Например, когда запись отмечается как выполненная, вы можете:
1. Переместить её в список "Завершенные"
2. Отправить уведомление по электронной почте клиенту
3. Создать задачу для последующих действий
4. Обновить пользовательское поле с датой завершения

## Настройка действий

Каждый тип действия имеет определенные параметры конфигурации:
- **Выбор цели**: Выберите, какие записи или пользователи затрагиваются
- **Условная логика**: Установите условия для выполнения действий
- **Сопоставление данных**: Используйте теги слияния для включения динамических данных из записей
- **Время выполнения**: Некоторые действия поддерживают задержки или запланированное выполнение

## Лучшие практики

1. **Сначала тестируйте**: Всегда тестируйте автоматизации с образцами данных перед активацией
2. **Начинайте с простого**: Начните с отдельных действий и постепенно добавляйте сложность
3. **Документируйте назначение**: Используйте понятные названия для описания того, что делает каждая автоматизация
4. **Мониторьте результаты**: Просматривайте журналы автоматизации для обеспечения ожидаемого поведения
5. **Избегайте конфликтов**: Будьте осторожны, чтобы не создать действия, которые отменяют друг друга

## Разрешения

Только администраторы проекта могут создавать и изменять действия автоматизации. Все участники команды могут просматривать активные автоматизации, чтобы понимать рабочий процесс проекта.
