檔案自訂欄位讓您能夠直接上傳檔案並將其附加到 Blue 記錄中。幾乎支援所有檔案類型，包括文件、試算表、圖片、PDF 等。

![File 1](/docs/File_1.png "File 1")

結構化檔案上傳確保所有人都能看到相關檔案

每個檔案欄位都支援上傳多個檔案，每個檔案大小可達 5GB。已上傳的檔案（如圖片、PDF 和 Word 文件）可在支援的檔案類型下進行預覽。所有專案使用者都可以下載附加檔案，以存取相關記錄資訊和內容。

![File 2](/docs/File_2.png "File 2")

使用者可以在檔案自訂欄位中預覽和下載檔案

上傳的檔案在專案範圍內共享，位於[「檔案」頁籤](/zh-TW/docs/projects/file-management)下。但是，具有受限記錄存取權限的自訂使用者角色將看不到這些檔案。專案檔案頁籤支援建立資料夾以整理檔案，但無法直接從自訂欄位內將檔案分類到這些資料夾中。

::callout
---

icon: lightbulb
---

不維護檔案版本歷史記錄 — 新上傳的檔案不會取代現有檔案。檔案也不包含在 CSV 匯出報告中。
::

## 表單整合

檔案自訂欄位[與表單無縫整合](/zh-TW/docs/projects/forms)，讓外部使用者在提交資訊時可以上傳檔案。例如，求職者可以在求職申請表單中附加履歷和求職信，客戶可以為創意專案要求提供圖片和範例，或者客戶可以透過客戶資料收集表單提交文件和資料。通過在表單中包含檔案自訂欄位，外部相關人員可以提供重要檔案和資產，以豐富從表單提交生成的記錄。這提供了收集內容的好處，無需外部使用者在 Blue 中擁有帳戶。

## API 檔案存取

如同任何其他系統檔案，透過檔案自訂欄位上傳的檔案可以透過 [Blue API](/zh-TW/docs/integrations/api) 進行存取和管理。當檔案使用檔案欄位附加到記錄時，這些檔案便可透過標準檔案相關 API 呼叫取得。開發人員可以利用完整的 API 功能套件，以程式化方式下載、上傳、整理和整合檔案自訂欄位中的檔案。無論是自動處理上傳的資產還是附加中繼資料，API 都能透過提供對 Blue 中所有使用者新增檔案的統一存取，實現無縫整合。

透過附加經常使用的文件和資產，檔案欄位消除了在多個位置搜尋與記錄相關內容的需要。這提供了對您所需檔案的快速、集中存取。
