Åtgärder är de operationer som Blue utför automatiskt när en automation utlöses. När ett utlösningsvillkor uppfylls, exekverar åtgärder specifika uppgifter för att effektivisera ditt arbetsflöde.

## Tillgängliga Åtgärder

Blue tillhandahåller en omfattande uppsättning åtgärder som kan kombineras för att skapa kraftfulla automationer:

### Uppgiftshanteringsåtgärder
- **Ändra Todo-lista**: Flytta en post till en annan lista inom projektet
- **Markera som Slutförd**: Markera automatiskt en post som klar
- **Markera som Ofullständig**: Återställ en post till ofullständig status
- **Kopiera Todo**: Skapa en kopia av posten med specificerade alternativ

### Tilldelningsåtgärder
- **Lägg till Tilldelad**: Tilldela en eller flera teammedlemmar till en post
- **Ta bort Tilldelad**: Ta bort specifika tilldelade från en post

### Datumhanteringsåtgärder
- **Ändra Förfallodatum**: Ställ in eller uppdatera förfallodatumet för en post
- **Ta bort Förfallodatum**: Rensa förfallodatumet från en post

### Organisationsåtgärder
- **Lägg till Tagg**: Applicera taggar för att kategorisera och organisera poster
- **Ta bort Tagg**: Ta bort specifika taggar från poster
- **Lägg till Färg**: Applicera färgkodning på poster för visuell organisation
- **Ta bort Färg**: Ta bort färgkodning från poster

### Checklista-åtgärder
- **Markera Checklisteobjekt som Klart**: Slutför specifika checklisteobjekt
- **Markera Checklisteobjekt som Inte Klart**: Återställ checklisteobjekt till ofullständiga
- **Skapa Checklista**: Lägg till nya checklistor med objekt till poster

### Anpassade Fältåtgärder
- **Lägg till Anpassat Fält**: Ställ in värden för anpassade fält på poster
- **Ta bort Anpassat Fält**: Rensa anpassade fältvärden

### Kommunikationsåtgärder
- **[Skicka E-post](/sv/docs/automations/actions/email-automations)**: Skicka automatiserade e-postmeddelanden med postdata
- **Generera PDF**: Skapa PDF-dokument från portabla dokumentmallar
- **Gör HTTP-förfrågan**: Integrera med externa tjänster via webhooks

## Hur Åtgärder Fungerar

Åtgärder kan kopplas samman för att skapa komplexa arbetsflöden. Till exempel, när en post markeras som slutförd, kan du:
1. Flytta den till en "Slutförd" lista
2. Skicka ett notifieringsmail till klienten
3. Skapa en uppföljningsuppgift
4. Uppdatera ett anpassat fält med slutförandedatumet

## Åtgärdskonfiguration

Varje åtgärdstyp har specifika konfigurationsalternativ:
- **Målval**: Välj vilka poster eller användare som påverkas
- **Villkorslogik**: Ställ in villkor för när åtgärder ska exekveras
- **Datamappning**: Använd sammanslagningstaggar för att inkludera dynamisk data från poster
- **Timing**: Vissa åtgärder stöder förseningar eller schemalagd exekvering

## Bästa Praxis

1. **Testa Först**: Testa alltid automationer med exempeldata innan aktivering
2. **Börja Enkelt**: Börja med enskilda åtgärder och lägg till komplexitet gradvis
3. **Dokumentera Syfte**: Använd tydlig benämning för att beskriva vad varje automation gör
4. **Övervaka Resultat**: Granska automationsloggar för att säkerställa förväntat beteende
5. **Undvik Konflikter**: Var försiktig med att inte skapa åtgärder som ångrar varandra

## Behörigheter

Endast Projektadministratörer kan skapa och modifiera automationsåtgärder. Alla teammedlemmar kan visa aktiva automationer för att förstå projektets arbetsflöde.
