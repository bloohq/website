---
title: Unternehmensabfrage
description: So rufen Sie Unternehmensinformationen mit der Blue API anhand von ID oder Slug ab.
---

Um Unternehmensinformationen abzurufen, können Sie die `company` Abfrage mit entweder der Unternehmens-ID oder dem Slug verwenden:

## Abfrage nach Unternehmens-ID

```graphql
query GetCompanyById {
  company(id: "company-id-here") {
    id
    name
    slug
    createdAt
    updatedAt
  }
}
```

## Abfrage nach Unternehmens-Slug

Die gleiche Abfrage akzeptiert auch einen Unternehmens-Slug, was es einfach macht, Unternehmensinformationen mit dem URL-freundlichen Identifikator abzurufen:

```graphql
query GetCompanyBySlug {
  company(id: "company-slug-here") {
    id
    name
    slug
    createdAt
    updatedAt
  }
}
```

## Beispielantwort

Beide Abfragen geben dasselbe Unternehmensobjekt zurück:

```json
{
  "data": {
    "company": {
      "id": "cuid123456789",
      "name": "Acme Corporation",
      "slug": "acme-corp",
      "createdAt": "2024-01-15T10:30:00.000Z",
      "updatedAt": "2024-03-20T14:45:00.000Z"
    }
  }
}
```

## Wichtige Hinweise

- Der `id` Parameter akzeptiert sowohl Unternehmens-IDs als auch Slugs
- Es werden nur Unternehmen zurückgegeben, bei denen der aktuelle Benutzer Mitglied ist
- Wenn das Unternehmen nicht gefunden wird oder der Benutzer keinen Zugriff hat, wird ein `CompanyNotFoundError` ausgelöst
- Gesperrte Unternehmen werfen ein `CompanyBannedError`

## Verfügbare Felder

Sie können nach zusätzlichen Unternehmensfeldern suchen, wie benötigt:

```graphql
query GetCompanyDetails {
  company(id: "company-slug") {
    id
    name
    slug
    # Add more fields as needed based on your requirements
    # Check the GraphQL schema for all available fields
  }
}
```
