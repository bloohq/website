---
title: Auslöser
description: 
---

Ein Auslöser ist ein Ereignis, das eine Automatisierung startet. Beispielsweise könnte ein Auslöser die Fertigstellung eines Datensatzes, eine Änderung des Verantwortlichen eines Datensatzes oder eine Änderung in einem benutzerdefinierten Feld sein. Wenn diese vordefinierte Bedingung erfüllt ist, wird die Automatisierung aktiviert.

Dies sind alle möglichen Automatisierungs-Auslöser in Blue:

- **Ein neuer Datensatz wird erstellt**: Löst aus, wenn ein neuer Datensatz zum Projekt hinzugefügt wird. Nützlich für die Initiierung von Folgeaktionen oder Benachrichtigungen im Zusammenhang mit neuen Aufgaben oder Einträgen. Dies löst auch aus, wenn es eine [neue Formularübermittlung](/de/docs/projects/forms) gibt.
- **Ein Datensatz wird in eine andere Liste verschoben**: Aktiviert sich, wenn ein Datensatz von einer Liste zu einer anderen innerhalb des Projekts übertragen wird, was eine Änderung in seiner Phase oder seinem Status anzeigt.
- **Ein Datensatz wird als abgeschlossen markiert**: Löst aus, wenn der Status eines Datensatzes auf 'abgeschlossen' aktualisiert wird und das Ende einer Aufgabe oder eines Prozesses signalisiert.
- **Ein Datensatz wird als unvollständig markiert**: Aktiviert sich, wenn ein zuvor als abgeschlossen markierter Datensatz zu einem unvollständigen Status zurückgesetzt wird.
- **Jemand wird einem Datensatz zugewiesen**: Löst aus, wenn ein neuer Verantwortlicher zu einem Datensatz hinzugefügt wird und seine Verantwortung für diese Aufgabe oder diesen Punkt anzeigt.
- **Jemand wird von einem Datensatz entfernt**: Aktiviert sich, wenn ein Verantwortlicher von einem Datensatz entfernt wird und eine Änderung in der Aufgabenverantwortung signalisiert.
- **Fälligkeitsdatum ändert sich**: Löst aus, wenn das Fälligkeitsdatum eines Datensatzes geändert wird, was für zeitkritische Aufgaben entscheidend sein kann.
- **Fälligkeitsdatum wird entfernt**: Aktiviert sich, wenn das Fälligkeitsdatum eines Datensatzes gelöscht wird, was möglicherweise eine Änderung in den Prioritäten oder Zeitplänen anzeigt.
- **Fälligkeitsdatum ist abgelaufen**: Löst aus, wenn das Fälligkeitsdatum eines Datensatzes ohne Fertigstellung verstreicht und potenzielle Verzögerungen oder Versäumnisse hervorhebt. Dies ist eine ausgezeichnete Automatisierung für Manager, da sie sich Datensätze/Aufgaben markieren können, wenn diese überfällig werden.
- **Tag wird zu Datensatz hinzugefügt**: Aktiviert sich, wenn ein neues Tag zu einem Datensatz hinzugefügt wird, nützlich für die Kategorisierung und Verfolgung von Änderungen in Aufgabenattributen.
- **Tag wird von Datensatz entfernt**: Löst aus, wenn ein Tag von einem Datensatz entfernt wird und eine Änderung in seiner Kategorisierung oder seinen Eigenschaften anzeigt.
- **Alle Checklisten werden als abgeschlossen markiert**: Aktiviert sich, wenn jedes Element in der Checkliste eines Datensatzes abgehakt wird und die Fertigstellung aller Teilaufgaben anzeigt.
- **Eine Checkliste wird als unvollständig markiert**: Löst aus, wenn ein Element in der Checkliste eines Datensatzes als unvollständig markiert wird und unfertige Teilaufgaben signalisiert.
- **Ein Datensatz wird von einem anderen Projekt verschoben/kopiert**: Aktiviert sich, wenn ein Datensatz von einem anderen Projekt in das aktuelle Projekt gebracht wird, hilfreich bei der Verfolgung projektübergreifender Arbeitsabläufe. Der Auslöser gilt sowohl, wenn der Datensatz manuell von einem Benutzer verschoben oder kopiert wird, als auch wenn er automatisch durch eine andere Automatisierung übertragen wird, und bietet eine umfassende Möglichkeit, projektübergreifende Aktivitäten zu verfolgen.
- **Ein benutzerdefiniertes Feld wird zu einem Datensatz hinzugefügt**: Löst aus, wenn Daten in ein spezifisches benutzerdefiniertes Feld in einem Datensatz geschrieben werden und eine Änderung oder Hinzufügung spezifischer Informationen anzeigt. Dies funktioniert nur mit Kontrollkästchen, Einfach- und Mehrfachauswahl benutzerdefinierten Feldern.
- **Ein benutzerdefiniertes Feld wird von einem Datensatz entfernt**: Aktiviert sich, wenn Daten aus einem spezifischen benutzerdefinierten Feld in einem Datensatz entfernt werden und eine Änderung in der Informationsstruktur des Datensatzes signalisiert.
- **Eine Farbe wird zu einem Datensatz hinzugefügt**: Löst aus, wenn eine vordefinierte Farbe einem Datensatz zugewiesen wird und Priorität, Fortschritt oder Kategorisierung anzeigt. Beispielsweise das Hinzufügen einer roten Farbe für hochpriorisierte Aufgaben oder einer grünen Farbe für laufende Elemente.
- **Eine Farbe wird von einem Datensatz entfernt**: Löst aus, wenn eine vordefinierte Farbe von einem Datensatz entfernt wird und eine Änderung im Status oder der Priorität signalisiert. Beispielsweise das Entfernen einer roten Farbe von einer Aufgabe, wenn sie nicht mehr hohe Priorität hat, oder das Entfernen einer grünen Farbe, wenn eine Aufgabe abgeschlossen ist.
- **Button für benutzerdefiniertes Feld geklickt**: Löst aus, wenn ein Benutzer auf ein Button-benutzerdefiniertes Feld in einem Datensatz klickt. Dies ermöglicht interaktive Arbeitsabläufe, bei denen Benutzer Automatisierungen manuell auslösen können, indem sie auf bestimmte Buttons innerhalb von Datensätzen klicken.
